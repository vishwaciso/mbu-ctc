What Happens When You Type a URL?

Deep Technical Breakdown of DNS → TCP → TLS → CDN → Load Balancer → Microservices → Database

1️⃣ DNS Resolution

Your browser first asks: "What is the IP address of this domain?"

The request travels through:

Browser → OS Resolver → ISP DNS → Root Server → TLD (.com) → Authoritative DNS

Finally, the domain resolves to an IP address.

2️⃣ TCP 3-Way Handshake

The browser establishes connection using:

SYN → SYN-ACK → ACK

This ensures reliable communication between client and server.

3️⃣ TLS / HTTPS Encryption

Secure communication begins with TLS handshake:

Client Hello → Server Hello → Certificate Exchange → Key Negotiation

Now all traffic is encrypted 🔐

4️⃣ CDN (Content Delivery Network)

Your request usually hits the nearest Edge server first.

This reduces latency and improves performance.

5️⃣ Load Balancer

Traffic is distributed across multiple backend servers.

Prevents overload and ensures high availability.

6️⃣ Microservices Architecture

Modern apps like YouTube are built with multiple services:

Auth Service | Recommendation Engine | Video Service | Ads Service

7️⃣ Database & Storage

Backend servers fetch data from distributed databases and object storage systems.

8️⃣ Response Sent Back

Server sends HTML + CSS + JavaScript.

Browser renders the page and starts streaming content.

User → DNS → TCP → TLS → CDN → Load Balancer → App Servers → Database → Response
